  "body_mdx": "## The Restart Tax\n\nAn agentic workflow that fails at step 15 and has no checkpoints pays a restart tax: every successful step before the failure gets repeated. The LLM calls, the tool calls, the API requests, the token cost. All of it runs again, from the beginning, to get back to where the failure occurred.\n\nFor short workflows, this is tolerable. For a 25-step research workflow that takes 15 minutes and costs $3 in tokens, repeating the first 14 steps to retry step 15 is a $2.80 tax on a $0.20 operation. Over hundreds of executions, this adds up significantly.\n\nCheckpointing eliminates the restart tax. After each successful step, the workflow saves its state. On failure, it resumes from the last checkpoint rather than the beginning.\n\n## What to Checkpoint\n\nA checkpoint captures everything needed to resume from a specific point in the workflow:\n\n- The workflow ID and current step number.\n- The conversation history (all messages exchanged with the LLM up to this point).\n- The outputs of all completed tool calls.\n- The current token budget consumption.\n- Any external state the workflow has modified (so the rollback logic knows what to undo).\n\n```python\nfrom dataclasses import dataclass\nfrom typing import Any\n\n@dataclass\nclass Checkpoint:\n    workflow_id: str\n    step: int\n    messages: list[dict]       # Full conversation history\n    tool_outputs: dict[str, Any]  # Keyed by idempotency key\n    tokens_spent: int\n    modified_resources: list[str]  # e.g. [\"db:report_sections:42\", \"email:pending:a3f9\"]\n    created_at: float\n```\n\n## Checkpoint Storage\n\nCheckpoints need to be durable. Storing them in memory defeats the purpose: if the process crashes, the checkpoint is gone.\n\nFor most workflows, a database table is sufficient:\n\n```sql\nCREATE TABLE workflow_checkpoints (\n    workflow_id    TEXT NOT NULL,\n    step           INT NOT NULL,\n    messages       JSONB NOT NULL,\n    tool_outputs   JSONB NOT NULL DEFAULT '{}',\n    tokens_spent   INT NOT NULL DEFAULT 0,\n    modified_resources JSONB NOT NULL DEFAULT '[]',\n    created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),\n    PRIMARY KEY (workflow_id, step)\n);\n\nCREATE INDEX ON workflow_checkpoints (workflow_id, step DESC);\n```\n\nOn failure, query for the latest checkpoint:\n\n```sql\nSELECT * FROM workflow_checkpoints\nWHERE workflow_id = $1\nORDER BY step DESC\nLIMIT 1;\n```\n\n## Writing Checkpoints\n\nThe checkpoint write happens immediately after a successful tool call, before the next LLM invocation:\n\n```python\nasync def run_workflow(workflow_id: str) -> WorkflowResult:\n    # Rollback handles the case where a failed workflow left external state in a partially modified condition.\n\nNot all operations can be rolled back. An email that was sent cannot be unsent. A payment that was charged cannot be reversed through a rollback. These are cases where idempotency (checking before acting) is the right mitigation. Rollback handles cases where the operation itself is reversible: a database record that was written, a file that was created, a reservation that was made.\n\n```python\nasync def rollback_workflow(workflow_id: str, to_step: int) -> RollbackResult:\n    checkpoint = await load_checkpoint(workflow_id, step=to_step)\n    if not checkpoint:\n        raise ValueError(f\"No checkpoint at step {to_step}\")\n\n    rolled_back = []\n    failed = []\n\n    # Roll back in reverse order: last modified first\n    for resource in reversed(checkpoint.modified_resources):\n        try:\n            await rollback_resource(resource)\n            rolled_back.append(resource)\n        except Exception as e:\n            failed.append({\"resource\": resource, \"error\": str(e)})\n\n    return RollbackResult(\n        rolled_back=rolled_back,\n        failed=failed,\n        to_step=to_step,\n    )\n\nasync def rollback_resource(resource_id: str) -> None:\n    kind, *parts = resource_id.split(\":\")\n    if kind == \"db\":\n        table, record_id = parts[0], parts[1]\n        await db.delete(table, record_id)\n    elif kind == \"file\":\n        path = parts[0]\n        await storage.delete(path)\n    # Extend with additional resource types as needed\n```\n\nThe `modified_resources` list in the checkpoint is how rollback knows what to undo.
