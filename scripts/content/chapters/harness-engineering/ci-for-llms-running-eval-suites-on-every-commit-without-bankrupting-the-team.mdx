Run the test suite, wait 5 minutes, get a pass or fail signal. The cost per run is negligible. Running it on every commit is obvious and universal.\n\nCI for LLM systems has a different cost profile. An eval suite with 500 examples against a frontier model costs real money per run. At $0.01 per example (including input and output tokens), 500 examples cost $5 per run. At 20 PRs per day across a team, that is $100 per day, $3000 per month, just for CI evals. Add end-to-end evals with agentic pipelines and the costs compound.\n\nThe naive response is to skip eval in CI entirely and rely on manual QA and production monitoring. This is how teams accumulate a backlog of silent regressions that only surface during user complaints.\n\nThe correct response is a tiered eval strategy that matches cost to frequency.\n\n## The Three Tiers\n\n**Tier 1: Smoke suite (every commit, every PR)**\n\n- 25 to 50 examples, carefully selected to cover the highest-risk scenarios\n- One example from each major input category in the golden dataset\n- Target runtime: under 90 seconds\n- Cost: under $0.50 per run\n- Gates: hard contract failures block merge; no metric thresholds at this tier\n\n**Tier 2: Regression suite (on merge to main, nightly)**\n\n- 200 to 300 examples, the core golden dataset\n- Full metric evaluation with thresholds\n- Target runtime: under 15 minutes\n- Cost: $2 to $5 per run\n- Gates: metric regressions beyond tolerance block deployment\n\n**Tier 3: Full suite (weekly, pre-release)**\n\n- Complete golden dataset plus edge cases\n- Human eval sampling for soft quality dimensions\n- Includes end-to-end agentic scenarios\n- Target runtime: 60 to 120 minutes\n- Cost: $20 to $50 per run\n- Output: comprehensive quality report for engineering and product leadership\n\n```\nFrequency:  Every PR      Merge to main    Weekly\nTier:       Smoke         Regression       Full\nExamples:   30-50         200-300          500+\nRuntime:    <90s          <15min           60-120min\nCost/run:   <$0.50        $2-5             $20-50\nBlocks:     Hard failures  Metric regressions  N/A (report only)\n```\n\n## Selecting Smoke Examples\n\nThe smoke suite is only valuable if it catches real regressions. Choosing the 30 to 50 examples requires deliberate selection:\n\n1. Include at least one example from every major input category\n2.
