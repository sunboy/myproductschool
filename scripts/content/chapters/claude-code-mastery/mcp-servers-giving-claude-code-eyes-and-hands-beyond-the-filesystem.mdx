Connect to Figma, Linear, Supabase, or your own internal APIs without leaving the conversation.", "body_mdx": "## What MCP Is\n\nMCP (Model Context Protocol) is a standard for connecting language model clients to external tools and data sources. It defines how a host (Claude Code) discovers, calls, and receives results from a server (Figma, Linear, a database, your internal API). The protocol handles schema advertisement, argument validation, and result formatting.\n\nFor Claude Code, MCP servers appear as additional tool categories alongside the built-in ones (Read, Edit, Bash, WebFetch). Claude Code can call them the same way: it sees a tool name, a parameter schema, and calls it with the relevant arguments.\n\n## The Architecture\n\n```mermaid\ngraph TD\n    CC[\"Claude Code (host)\\nRead, Edit, Bash, WebFetch\"]\n    CC -->|\"stdio / HTTP\"| FM[\"Figma MCP server\\nmcp__figma__get_design_context\\nmcp__figma__get_screenshot\"]\n    CC -->|\"stdio / HTTP\"| LM[\"Linear MCP server\\nmcp__linear__get_issue\\nmcp__linear__save_issue\"]\n    CC -->|\"stdio / HTTP\"| SM[\"Supabase MCP server\\nmcp__supabase__execute_sql\"]\n    CC -->|\"stdio / HTTP\"| CM[\"Custom MCP server\\nmcp__your_api__any_endpoint\"]\n    style CC fill:#0d2e2e,color:#fff\n    style FM fill:#14b8a6,color:#fff\n    style LM fill:#14b8a6,color:#fff\n    style SM fill:#14b8a6,color:#fff\n    style CM fill:#374151,color:#fff\n```\n\nMCP servers run as separate processes. They communicate with Claude Code over stdio or HTTP. Claude Code discovers their available tools at session start and includes them in its tool namespace.\n\n## Connecting an MCP Server\n\nMCP servers are configured in `.claude/settings.json` or `~/.claude/settings.json`:\n\n```json\n{\n  \"mcpServers\": {\n    \"supabase\": {\n      \"command\": \"npx\",\n      \"args\": [\"-y\", \"@supabase/mcp-server-supabase@latest\"],\n      \"env\": {\n        \"SUPABASE_URL\": \"https://your-project.supabase.co\",\n        \"SUPABASE_SERVICE_ROLE_KEY\": \"your-key\"\n      }\n    },\n    \"linear\": {\n      \"command\": \"npx\",\n      \"args\": [\"-y\", \"@linear/mcp-server@latest\"],\n      \"env\": {\n        \"LINEAR_API_KEY\": \"your-key\"\n      }\n    },\n    \"figma\": {\n      \"type\": \"http\",\n      \"url\": \"https://mcp.figma.com/mcp\"\n    }\n  }\n}\n```\n\nAfter saving this, restart Claude Code. The next session will include all configured MCP tools in its available tool set.\n\n## What You Can Do With MCP\n\nThe practical applications fall into three categories:\n\n**Read from external sources.** `mcp__figma__get_design_context` fetches a Figma screen's layout, colors, and component structure. `mcp__linear__get_issue` fetches the full details of a ticket including comments and attachments. `mcp__supabase__execute_sql` runs a query and returns results. Claude Code can use these to make decisions without you copy-pasting information into the conversation.\n\n**Write to external systems.** `mcp__linear__save_issue` creates or updates a ticket. `mcp__supabase__execute_sql` can run insert and update statements (with appropriate permissions). The action the model decides to take can propagate directly to the external system, no copy-paste required.\n\n**Trigger workflows.** Some MCP servers wrap complex operations. A CI/CD MCP server might expose `trigger_build`, `get_build_status`, `rollback_deployment`. Claude Code can orchestrate these as part of a larger task.\n\n## Building a Custom MCP Server\n\nFor internal APIs, you build your own. The MCP SDK has implementations in TypeScript and Python. A minimal server that exposes one tool:\n\n```typescript\nimport { Server } from '@modelcontextprotocol/sdk/server/index.js'\nimport { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js'\n\nconst server = new Server(\n  { name: 'my-internal-api', version: '1.0.0' },\n  { capabilities: { tools: {} } }\n)\n\nserver.setRequestHandler('tools/list', async () => ({\n  tools: [{\n    name: 'get_feature_flag',\n    description: 'Get the current state of a feature flag',\n    inputSchema: {\n      type: 'object',\n      properties: {\n        flag_name: { type: 'string', description: 'The flag identifier' }\n      },\n      required: ['flag_name']\n    }\n  }]\n}))\n\nserver.setRequestHandler('tools/call', async (request) => {\n  if (request.params.name === 'get_feature_flag') {\n    const { flag_name } = request.params.arguments as { flag_name: string }\n    const result = await fetchFlagFromYourAPI(flag_name)\n    return { content: [{ type: 'text', text: JSON.stringify(result) }] }\n  }\n})\n\nawait server.connect(new StdioServerTransport())\n```\n\nThis server, registered in settings.json, makes `mcp__my-internal-api__get_feature_flag` available in every Claude Code session. Claude Code will call it when it needs flag information, without you manually fetching and pasting it.\n\n## MCP and Tool Permissions\n\nMCP tool calls appear in the permission system like built-in tools. You can allowlist them:\n\n```json\n{\n  \"permissions\": {\n    \"allow\": [\n      \"mcp__supabase__execute_sql\",\n      \"mcp__linear__get_issue\",\n      \"mcp__linear__list_issues\",\n      \"mcp__figma__get_design_context\",\n      \"mcp__figma__get_screenshot\"\n    ],\n    \"deny\": [\n      \"mcp__supabase__delete_*\"\n    ]\n  }\n}\n```\n\nThe wildcard pattern `mcp__supabase__*` permits all Supabase MCP tools. Be specific about write operations. Read tools are generally safe to allowlist. Write and delete tools should be explicit.\n\n## When MCP Changes the Workflow\n\nConsider a typical design-to-code workflow without MCP:\n\n1. Open Figma, find the screen, inspect it\n2. Copy hex colors, layout values, component names\n3. Paste into Claude Code conversation\n4. Ask Claude Code to implement it\n5. Repeat for every ambiguity\n\nWith the Figma MCP server:\n\n1. Paste the Figma URL into Claude Code\n2. Claude Code calls `mcp__figma__get_design_context` with the node ID\n3. It gets the full layout, tokens, and component structure\n4. It implements the screen\n\nThe human copy-paste loop is gone. Claude Code can access the source of truth directly. The same pattern applies to Linear tickets, database schemas, API documentation, internal wikis. Any external system you connect via MCP becomes directly accessible in the conversation."}
