When you mark a breakpoint with `cache_control: { type: 'ephemeral' }`, you're telling the API: \"everything up to this point is stable, compute it once, serve it from memory on future requests.\"\n\nThe read cost is $0.30/MTok, versus $3.00/MTok for fresh processing. Volatile content goes at the bottom. Commits to the stable prefix cost money. Commits that only change the tail are cheap.\n\n```mermaid\ngraph TD\n    A[\"TOOLS<br/>(function definitions, rarely change)\"]\n    B[\"SYSTEM PROMPT<br/>(persona, rules, format instructions)\"]\n    C[\"DOCUMENTS / CONTEXT<br/>(retrieved chunks, reference material)\"]\n    D[\"CONVERSATION HISTORY<br/>(prior turns)\"]\n    E[\"USER MESSAGE<br/>(current request, always fresh)\"]\n\n    A --> B --> C --> D --> E\n\n    style A fill:#2d6a4f,color:#fff\n    style B fill:#2d6a4f,color:#fff\n    style C fill:#52b788,color:#fff\n    style D fill:#95d5b2,color:#000\n    style E fill:#d8f3dc,color:#000\n```\n\nGreen = stable, cache here. Light = volatile, never cache here.\n\nProjectDiscovery, a security scanning platform processing thousands of vulnerability reports daily, built their system around this pattern. Anthropic allows up to 4 explicit cache breakpoints per request. Higher request rates mean more cache reuse within the 5-minute window, which means the latency benefit applies to a larger fraction of your traffic.\n\n## Cache Lifetime Strategy\n\nThe 5-minute TTL is the default. These never change between users or requests.\n\n**Cache if stable**: Large reference documents, API documentation, codebases, policy documents. These change rarely. When they do change, the cache invalidates naturally.\n\n**Never cache**: User messages, session identifiers, per-user profile data injected into the prompt, real-time context like current time or live data fetches. These belong at the tail, after the last cache breakpoint.\n\nOne subtlety: thinking blocks cannot be marked with `cache_control` directly, but they cache correctly when they appear in previous assistant turns in the conversation history.
