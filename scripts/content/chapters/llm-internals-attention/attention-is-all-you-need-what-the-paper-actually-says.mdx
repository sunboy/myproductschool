{"hook_text": "The 2017 transformer paper replaced recurrence with a single mechanism: every position attends to every other position simultaneously. That architectural choice explains both why transformers scaled so well and why long context is expensive.", "body_mdx": "The transformer paper from Vaswani et al. arrived in 2017 as a machine translation paper. Its title, \"Attention Is All You Need,\" reads like a provocation. It turns out to be a precise description.\n\nBefore transformers, sequence models processed tokens one at a time. RNNs passed a hidden state forward through the sequence. LSTMs added gating to help the hidden state survive long sequences. Both architectures had a fundamental problem: the gradient signal from token 500 back to token 1 had to flow through 499 intermediate steps. Long-range dependencies were structurally hard to learn.\n\nTransformers eliminated the sequential bottleneck entirely. Every position attends directly to every other position in a single layer. The paper called this \"self-attention.\"\n\n## What Self-Attention Actually Computes\n\nGiven an input sequence of $n$ token embeddings, each of dimension $d_{model}$, self-attention produces three linear projections: queries ($Q$), keys ($K$), and values ($V$).\n\n```\nQ = X * W_Q    # shape: (n, d_k)\nK = X * W_K    # shape: (n, d_k)\nV = X * W_V    # shape: (n, d_v)\n```\n\nThe attention output for the sequence is:\n\n```\nAttention(Q, K, V) = softmax(Q * K^T / sqrt(d_k)) * V\n```\n\nBreaking this down:\n\n1. `Q * K^T` produces an $(n \\times n)$ matrix of dot products. Entry $(i, j)$ measures how much position $i$'s query aligns with position $j$'s key.\n2. Dividing by $\\sqrt{d_k}$ prevents the dot products from growing too large, which would push softmax into regions where gradients vanish.\n3. `softmax(...)` normalizes the rows into probability distributions. Each row sums to 1.\n4. Multiplying by $V$ takes a weighted average of the value vectors. Each output position is a blend of all value vectors, weighted by attention scores.\n\nThis is a differentiable, learned retrieval mechanism. The model learns $W_Q$, $W_K$, and $W_V$ jointly with everything else during training.\n\n## Why the Quadratic Cost Matters\n\nThe $Q * K^T$ step is $O(n^2)$ in sequence length. For $n = 4096$ tokens, that's 16 million attention scores per head. For $n = 128000$ tokens (Claude's 128k context), that's 16 billion scores, per head, per layer.\n\nThis is not a software optimization problem. It's geometry. You have $n$ query vectors and $n$ key vectors, and every pair must be scored.\n\nThe paper's original model used $n = 512$ tokens. GPT-3 used $n = 2048$. When OpenAI released the 128k context window for GPT-4 Turbo in late 2023, the quadratic cost meant attention computation alone increased by a factor of $(128000/2048)^2 \\approx 3900x$ relative to a 2k context.\n\nPractitioners handle this several ways:\n- **Sparse attention** (Longformer, BigBird): attend to local windows plus a few global tokens, reducing cost to $O(n \\log n)$ or $O(n)$\n- **Flash Attention**: a CUDA kernel that tiles the QK^T computation to stay in GPU SRAM, reducing memory bandwidth but not FLOPs\n- **Sliding window attention** (Mistral): each token attends only to the previous 4096 tokens\n- **Linear attention** (Mamba, RWKV): reformulate the attention mechanism to be recurrent at inference time\n\nNone of these match full attention's quality across all tasks. Full attention remains the baseline that others approximate.\n\n## Multi-Head Attention\n\nThe paper ran attention not once but $h$ times in parallel, with separate projection matrices:\n\n```\nMultiHead(Q, K, V) = Concat(head_1, ..., head_h) * W_O\n\nwhere head_i = Attention(Q * W_Q_i, K * W_K_i, V * W_V_i)\n```\n\nIn the original paper, $h = 8$ and $d_k = d_v = d_{model}/h = 64$. The intuition: different heads specialize in different relationship types. One head might track syntactic agreement. Another might track coreference. Empirical studies of BERT attention patterns found heads that reliably attend to the previous token, to sentence boundaries, to the subject of a sentence.\n\nModern models use more heads. GPT-3 175B uses 96 heads. Llama 3 70B uses 64 heads with grouped-query attention, where multiple query heads share a single key/value head, reducing KV cache size.\n\n## The Full Architecture\n\nSelf-attention is one component inside a transformer block:\n\n```mermaid\ngraph TD\n    A[Input Embeddings + Positional Encoding] --> B[Multi-Head Self-Attention]\n    B --> C[Add & Layer Norm]\n    C --> D[Feed-Forward Network]\n    D --> E[Add & Layer Norm]\n    E --> F[Output to Next Layer]\n```\n\nThe feed-forward network (FFN) is two linear layers with a nonlinearity between them, applied independently to each position:\n\n```\nFFN(x) = max(0, x * W_1 + b_1) * W_2 + b_2\n```\n\nThe FFN dimension is typically 4x the model dimension. In GPT-3, $d_{model} = 12288$, $d_{ffn} = 49152$. The FFN holds the majority of the model's parameters.\n\n\"Add & Norm\" refers to the residual connection (adding the pre-attention input back to the post-attention output) plus layer normalization. Residual connections let gradients flow directly through the network during training, which is why transformers can stack 96 layers without vanishing gradients.\n\n## What \"Decoder-Only\" Means\n\nThe original paper had an encoder-decoder architecture: the encoder processes the source sequence with bidirectional attention, the decoder generates the target sequence attending to the encoder output and to its own previous tokens with causal masking.\n\nGPT and its successors dropped the encoder. A decoder-only model processes all input tokens together, but with a causal mask: position $i$ can only attend to positions $\\leq i$. This prevents the model from \"cheating\" by looking at future tokens during training.\n\nAt inference time, causal masking is automatically satisfied because future tokens don't exist yet. The KV cache works precisely because of this: once a key and value have been computed for position $i$, they never change as new tokens are generated.\n\nBERT-style models (encoder-only) use bidirectional attention without a causal mask, which makes them better for classification and embedding tasks but unable to generate text autoregressively.\n\n## Positional Encoding\n\nSelf-attention has no inherent notion of position. Without additional signal, the model treats the sequence as a bag of tokens. The original paper added fixed sinusoidal positional encodings:\n\n```\nPE(pos, 2i) = sin(pos / 10000^(2i/d_model))\nPE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))\n```\n\nModern models use rotary positional embeddings (RoPE, used in Llama and Mistral) or ALiBi (used in some Falcon variants). RoPE encodes position as a rotation applied to query and key vectors, which gives better length generalization than fixed sinusoids and is central to how models extend beyond their training context length.\n\nThe architecture that replaced RNNs in 2017 is now the substrate for everything from GPT-4 to protein structure prediction to image generation. Understanding the quadratic cost of attention is not trivia. It explains every architectural choice in LLM deployment: why KV caches exist, why MoE models route tokens sparsely, why quantization matters, and why \"long context\" is a genuinely hard engineering problem."}
