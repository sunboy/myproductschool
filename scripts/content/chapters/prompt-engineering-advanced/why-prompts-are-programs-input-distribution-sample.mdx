{"hook_text": "A prompt is a prefix that shifts the probability distribution over next tokens. Understanding this makes it obvious why small wording changes have large effects.", "body_mdx": "## The Mental Model That Changes Everything\n\nWhen you write a prompt, you are not issuing a command. You are providing a prefix to a function that outputs a probability distribution over every possible next token. The model samples from that distribution. The prompt you write determines the shape of that distribution.\n\nThis is not a metaphor. This is the literal mechanism.\n\n```\nP(next_token | prompt) = softmax(logits)\n```\n\nEvery word you add, move, or remove shifts `P(next_token | prompt)`. Some shifts are small. Some are catastrophic. The model does not interpret your *intent*. It computes conditional probability over its training distribution.\n\n## What \"Input → Distribution → Sample\" Actually Means\n\nBreak it into three stages:\n\n**Input**: The full context window, including system prompt, user turn, and any injected content. The model sees this as a flat token sequence.\n\n**Distribution**: The forward pass produces logit scores for every token in the vocabulary. After temperature and top-p filtering, you get a probability distribution.\n\n**Sample**: One token is drawn. That token is appended to the context. The process repeats until a stop condition fires.\n\n```mermaid\nflowchart LR\n    A[\"Prompt (context)\"] --> B[\"Transformer forward pass\"]\n    B --> C[\"Logit scores per vocab token\"]\n    C --> D[\"Temperature + top-p filter\"]\n    D --> E[\"Sample one token\"]\n    E --> F[\"Append to context\"]\n    F --> B\n```\n\nNothing in this loop cares what you *meant*. It cares what sequence of tokens is statistically likely given the prefix.\n\n## Why Small Wording Changes Have Large Effects\n\nBecause the distribution is non-linear. Adding one token changes every subsequent conditional probability. Consider:\n\n- `\"Summarize this article.\"`: broad distribution, many plausible summaries\n- `\"Summarize this article in three bullet points.\"`: distribution sharply concentrated on bullet-format outputs\n- `\"Summarize this article in three bullet points. Be concise.\"`: distribution further shifted toward short bullets\n\nEach word is an additional constraint on the conditional. The distribution narrows (or widens, or shifts) with each addition.\n\nThis is also why the same underlying instruction written two different ways can produce wildly different outputs. The surface form of the text, its exact token sequence, is what the model conditions on. Paraphrasing is not neutral.\n\n## Token Probability and the Temperature Knob\n\nTemperature (`τ`) scales the logits before the softmax:\n\n```\nP(token_i) = exp(logit_i / τ) / sum_j(exp(logit_j / τ))\n```\n\nAt `τ = 1.0`, you get the model's native distribution.\nAt `τ < 1.0`, the distribution sharpens, and high-probability tokens become more dominant.\nAt `τ > 1.0`, the distribution flattens, giving low-probability tokens more chances.\n\nFor deterministic, factual tasks, lower temperature reduces variance. For creative tasks, higher temperature lets the model explore lower-probability paths. Setting temperature without understanding the target distribution is guessing.\n\n## What Training Data Has to Do With It\n\nThe distribution `P(next_token | prompt)` reflects the training corpus. If a phrase pattern appeared frequently in training data followed by certain kinds of completions, the model has learned that association.\n\nThis means:\n\n- Prompts that resemble patterns from high-quality training sources (e.g., technical documentation, academic papers) shift the distribution toward that register.\n- Prompts that resemble low-quality or off-topic patterns produce noisier outputs.\n- Domain-specific jargon acts as a signal. It narrows the distribution toward responses that match the domain.\n\n```\n\"The root cause of the latency regression is\" → high P(technical explanation)\n\"The vibes were off because\" → high P(casual, qualitative response)\n```\n\nThis is why \"write like a staff engineer\" in a system prompt actually changes output quality, not because the model obeys the instruction, but because the phrase pattern shifts the token distribution toward a different register.\n\n## Practical Consequence: Prompts Are Sensitive to Order\n\nBecause each token conditions all subsequent tokens, the order of information in a prompt matters. In a long context, tokens closer to the generation point have stronger influence. The model's attention distribution is not flat across the context window.\n\nThis creates a design principle: put the most important constraints and examples *late* in the prompt, close to where generation begins. This is why few-shot examples at the end of a prompt outperform the same examples at the beginning.\n\n## What This Changes About How You Debug Prompts\n\nWhen a prompt produces a bad output, the question is never \"why didn't the model understand me?\" It is always: \"what did I add to the probability distribution that made this output more likely than the output I wanted?\"\n\nDebugging prompts is distribution debugging. The interventions are:\n\n1. Add tokens that make the desired output more probable (examples, constraints, format instructions).\n2. Remove tokens that make undesired outputs more probable (ambiguous phrasing, contradictory instructions).\n3. Adjust temperature to exploit or reduce variance in the distribution.\n\nThere is no other lever. You do not have access to the weights. You only control the prefix."}
