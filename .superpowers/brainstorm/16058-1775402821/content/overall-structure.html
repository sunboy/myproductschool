Where should Product Autopsy live?

Currently it's nested under /explore/showcase. The spec calls for a top-level /autopsy route with its own immersive dark theme. Which approach fits better?

A

Own top-level route — /autopsy

Dedicated dark-themed immersive experience. Completely separate from the main app shell (no NavRail/TopBar from the (app) layout). Feels like a distinct "mode" of the product.

Pros

  • Full creative control over dark theme + ambient effects
  • Matches the editorial magazine feel of the spec
  • No design conflict with the main light Terra theme

Cons

  • Migration cost — existing /explore/showcase routes move
  • Less integrated with main nav and progress
B

Stay in (app) layout — /explore/autopsy

Keep the existing routing under /explore. The autopsy content lives inside the main app shell with NavRail. Dark background applied only to the content area, not the shell.

Pros

  • No routing migration needed
  • Consistent navigation — user always has NavRail
  • Progress visible alongside other features

Cons

  • Shell conflicts with dark editorial aesthetic
  • Less immersive — always feels like a sub-page
C

Hybrid — /autopsy but with back-to-app link

Top-level /autopsy route with its own dark layout. A small persistent "← Back to HackProduct" pill in the corner keeps connection to the main app. Best of both.

Pros

  • Full dark immersive design freedom
  • User doesn't feel lost — always a way back
  • Matches the spec's navigation model exactly

Cons

  • Migration cost (same as A)
  • Slightly more layout code to manage
