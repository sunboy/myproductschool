Implementation Approaches — Product Autopsy

Three ways to structure the build. All share the same routing decision (keep /explore/showcase, use app shell).

A

Full rebuild — new content schema + new components

Introduce the spec's StorySection JSON schema and story data model alongside the existing decision/challenge schema. Build all section layout components (cover, split_panel, stat, before_after, principle, CTA). Build 4-5 animated illustration components. Existing showcase grid/cards stay largely intact — we add the story reader as a new route /explore/showcase/[slug]/stories/[storySlug]. Challenges link out to /workspace/challenges/[id].

  • ✅ Matches spec exactly, very extensible
  • ✅ Clean separation: stories vs challenges vs grid
  • ✅ Illustrations are fully data-driven and reusable
  • ⚠️ Most code to write — 15-20 new files
  • ⚠️ Need to seed story content JSON for Notion
B

Recommended — Evolve existing + add story reader

Keep the existing grid + detail + challenge card components. Add the story reader as a new route. Extend the autopsy_decisions table with a story_sections JSONB column so each decision can optionally have a narrative. The story reader is a new scroll-based page component. Challenge links from the story CTA go to the existing showcase detail page (not external workspace). Illustrations are a separate feature-flagged layer.

  • ✅ Least disruption to working code
  • ✅ Story content lives in the same DB row as the decision
  • ✅ Feature flag NEXT_PUBLIC_ILLUSTRATION_MODE=animated|static controls fidelity
  • ✅ ~10-12 new files, most existing files untouched
  • ⚠️ Decision-level story (1 story per decision) vs product-level story (1 story per product)
C

Thin slice — story reader only, no illustrations yet

Build the story reader with all layout section components but use a static IllustrationPlaceholder for all illustrations. Add illustrations in a follow-up sprint. This gets the editorial experience shipped fast and validates the layout system before investing in animation.

  • ✅ Fastest to ship
  • ✅ Can validate story UX without animation complexity
  • ⚠️ Placeholder illustrations feel unfinished
  • ⚠️ Doesn't deliver on the "killer" goal yet

My recommendation

Option B — evolve existing infrastructure, add product-level stories (not decision-level), and build illustrations with the feature flag. Stories live at /explore/showcase/[slug]/stories/[storySlug], seeded for Notion first.
